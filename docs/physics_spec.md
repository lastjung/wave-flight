# Wave Flight 물리 및 좌표 명세서

본 문서는 이 게임에서 사용하는 Three.js 기반 좌표계와 플레이어 조작, 지형 이동 로직을 정의합니다.

## 1. 좌표축 정의 (Three.js 표준)

| 축      | 방향 (+)          | 방향 (-)         | 게임 내 역할                  |
| :------ | :---------------- | :--------------- | :---------------------------- |
| **X축** | 오른쪽 (Right)    | 왼쪽 (Left)      | 비행기 좌우 이동              |
| **Y축** | 위 (Up)           | 아래 (Down)      | 비행기 상하 이동 및 지형 높이 |
| **Z축** | **뒤 (Backward)** | **앞 (Forward)** | 전진 및 거리 (깊이)           |

## 2. 플레이어 조작 로직

- **좌우 제어 (X축):**
  - `A` / `Left Arrow`: `-X` 방향으로 이동 (좌측).
  - `D` / `Right Arrow`: `+X` 방향으로 이동 (우측).
  - 이동량에 따라 기체가 Z축을 중심으로 회전(Roll)하여 뱅킹 효과를 줌.
- **상하 제어 (Y축):**
  - `W` / `Up Arrow`: `-Y` 방향으로 고도 하강 (아래).
  - `S` / `Down Arrow`: `+Y` 방향으로 고도 상승 (위).
  - 조종 시 기체 코(Nose)가 들리거나 숙여지는 Pitch 회전 발생.

## 3. 지형 이동 및 전진 로직

- **전진의 정의:** 플레이어는 논리적으로 **`-Z` 방향(화면 안쪽)**으로 전진합니다.
- **상대적 지형 이동:** 플레이어가 앞으로 가기 위해서는 지형 무늬가 플레이어에게 **`+Z` 방향(화면 바깥쪽)**으로 다가와야 합니다.
- **노이즈 샘플링 공식:** `(z + travel)`
  - `travel`은 시간과 속도(`speed` 슬라이더)의 곱입니다.
  - 이 공식에 의해 지평선(`-Z`)에 있던 노이즈 패턴이 시간이 지남에 따라 플레이어(`0` 또는 `+Z`) 쪽으로 밀려 내려오게 됩니다.
- **속도 제어:**
  - 현재는 UI의 `speed` 슬라이더를 통해 속도를 조절합니다.
  - 추후 이 속도 값에 난이도 가중치를 곱하여 게임 진행 정도에 따라 빨라지도록 설계합니다.

## 4. 단위 시스템 (Unit System)

이 게임의 가상 세계는 Three.js의 추상화된 단위를 사용하며, 논리적으로 다음과 같이 설계되었습니다.

### 4.1. 거리 및 크기 (World Units)

- **추상 단위:** 가상 공간의 `1.0` 수치는 1단위를 의미합니다.
- **실제 비율 가이드:** 비행기의 몸체 길이(약 2.8)를 대략 **3미터** 정도로 가정하면 시각적 밸런스가 적절합니다.
- **주요 수치:**
  - 비행기 날개폭: 약 4.4 단위
  - 지형 크기: 34 x 110 단위 (기체 길이의 약 40배)
  - 초기 Z 좌표: `5.0` (관찰자 쪽으로 기체 약 2배 길이만큼 후퇴한 위치)

### 4.2. 시간 (Time Units)

- **기준:** **초 (Seconds)**
- **프레임 계산:** 매 프레임의 간격(`dt`)을 초 단위로 계산하여 이동 속도와 애니메이션을 동기화합니다.
- **예시:** 속도가 1.0일 때, 지형은 초당 약 12단위 이동합니다.

### 4.3. 각도 (Angle Units)

- **기준:** **라디안 (Radians)**
- **참고:** 브라우저 시스템상 `Math.PI` (180도)를 기준으로 회전값을 계산합니다.
